# S3 (Simple Storage Service)

- Object storage with 99.999999999% (11 nines) durability; stores files as objects in buckets.
- Bucket names are globally unique; objects are accessed by key (path-like, but flat — no real directories).
- Storage classes, lifecycle policies, and versioning optimize cost and protect data.

# Core Building Blocks

### Buckets and Objects

- **Bucket**: Container for objects; globally unique name; created in a specific region.
- **Object**: File + metadata; key = path (e.g. `logs/2024/app.log`); max size 5 TB.
- **Prefix**: Simulates directories (`logs/`, `images/`); no actual hierarchy.
- **Multipart upload**: Required for objects > 5 GB; recommended for > 100 MB.

### Storage Classes

| Class | Use Case | Cost | Retrieval |
|-------|----------|------|-----------|
| Standard | Frequently accessed | Highest | Instant |
| Standard-IA | Infrequent access | Lower storage, retrieval fee | Instant |
| One Zone-IA | Infrequent, non-critical | Lower (single AZ) | Instant |
| Intelligent-Tiering | Unknown access pattern | Auto-moves between tiers | Instant |
| Glacier Instant | Archive, instant access | Low storage | Instant |
| Glacier Flexible | Archive | Very low | Minutes to hours |
| Glacier Deep Archive | Long-term archive | Lowest | 12-48 hours |

### Versioning

- Keeps all versions of an object; protect from accidental overwrite/delete.
- Delete creates a "delete marker" (recoverable); permanently delete a specific version to truly remove.
- **MFA Delete**: Require MFA to permanently delete versions or disable versioning.

### Lifecycle Policies

```json
{
  "Rules": [{
    "ID": "archive-old-logs",
    "Filter": { "Prefix": "logs/" },
    "Transitions": [
      { "Days": 30, "StorageClass": "STANDARD_IA" },
      { "Days": 90, "StorageClass": "GLACIER" }
    ],
    "Expiration": { "Days": 365 }
  }]
}
```

### Bucket Policy (Resource Policy)

```json
{
  "Statement": [{
    "Effect": "Allow",
    "Principal": { "AWS": "arn:aws:iam::123456789012:role/app" },
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::my-bucket/*"
  }]
}
```

- Bucket policy controls access at the bucket level (cross-account, public access).
- **Block Public Access**: Account and bucket-level setting; enable by default.

### Access Control

| Method | Scope | Use Case |
|--------|-------|----------|
| IAM policy | Identity | Internal access from same account |
| Bucket policy | Bucket | Cross-account, public, conditions |
| ACL | Object/Bucket | Legacy; avoid; use policies instead |
| Block Public Access | Account/Bucket | Safety net against accidental public |

### Common CLI Commands

```bash
aws s3 ls                           # list buckets
aws s3 ls s3://my-bucket/prefix/    # list objects
aws s3 cp file.txt s3://my-bucket/  # upload
aws s3 sync ./dir s3://my-bucket/   # sync directory
aws s3 rb s3://my-bucket --force    # delete bucket and contents
```

# Troubleshooting Guide

### "Access Denied" on S3 operation
1. Check IAM policy: does the identity have the required S3 action?
2. Check bucket policy: does it allow or deny the caller?
3. Check Block Public Access settings.
4. Check object ownership: objects uploaded by another account may be owned by that account.

### Objects not transitioning to Glacier
1. Check lifecycle rule: correct prefix filter, days, and status (Enabled).
2. Objects must be > 128 KB for IA/Glacier transitions.
3. Check lifecycle rule isn't limited by tags that don't match.

### Versioning enabled but can't recover deleted file
1. Delete creates a delete marker; the old versions still exist.
2. List versions: `aws s3api list-object-versions --bucket my-bucket --prefix key`.
3. Delete the delete marker to restore, or copy a specific version.

# Quick Facts (Revision)

- S3 durability: 99.999999999% (11 nines); availability varies by class.
- Bucket names are globally unique across all AWS accounts.
- Versioning protects against accidental deletion; enable on important buckets.
- Block Public Access is a safety net — enable at account level.
- S3 is eventually consistent for overwrite PUTs and DELETEs (strong read-after-write for new PUTs since 2020).
- Lifecycle policies automate transitions between storage classes and expiration.
- Use `s3 sync` for efficient directory uploads; only transfers changed files.
- Objects > 5 GB must use multipart upload.
