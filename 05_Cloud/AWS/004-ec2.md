# EC2 (Elastic Compute Cloud)

- Virtual machines on AWS; choose instance type (CPU/memory), AMI (OS image), and storage (EBS volumes).
- User data scripts run at first boot for automated configuration; instance profiles provide IAM role credentials.
- Lifecycle: Running (billed) → Stopped (EBS billed) → Terminated (deleted); Spot instances for interruptible workloads.

# Core Building Blocks

### Instance Types

- **Naming**: `family.generation.size` (e.g. `t3.micro`, `m5.large`).
- **Families**: `t` (burstable), `m` (general), `c` (compute), `r` (memory), `g` (GPU), `i` (storage I/O).
- **Size**: micro → small → medium → large → xlarge → 2xlarge...
- `t3.micro`: 2 vCPU burstable, 1 GiB RAM; free tier eligible.
- Instance type naming: `family.generation.size` (e.g. `t3.micro`).
- `t` family uses CPU credits; exhausting credits causes throttling.

| Family | Optimized For | Example Use |
|--------|--------------|-------------|
| t3 | Burstable general | Dev/test, small apps |
| m5 | Balanced | Web servers, app servers |
| c5 | Compute | Batch, CI/CD, encoding |
| r5 | Memory | Databases, caching |
| g4 | GPU | ML inference, graphics |

### AMI (Amazon Machine Image)

- Template containing root volume snapshot, OS, and config.
- Sources: Amazon (Amazon Linux, Ubuntu), Marketplace, or custom (from instance/snapshot).
- Region-specific; copy to other regions for multi-region deployments.

### EBS (Elastic Block Store)

- Block storage attached to EC2 (same AZ); persists independently from instance.
- **Types**: `gp3` (general SSD, baseline 3000 IOPS), `io2` (high IOPS), `st1`/`sc1` (HDD throughput).
- **Snapshots**: Point-in-time backup stored in S3; copy across regions; create new volumes from snapshots.
- Root volume: deleted on terminate by default; additional volumes: persist by default.
- EBS volumes are AZ-specific; use snapshots to move data across AZs/regions.

### User Data and Metadata

- **User data**: Shell script or `cloud-init` config; runs once at first boot (base64-encoded).
- **IMDS** (Instance Metadata Service): `http://169.254.169.254/latest/meta-data/` — instance ID, AMI, role creds.
- **IMDSv2**: Token-based (more secure); `PUT` for token, then `GET` with header.
- User data runs once at first boot; use `cloud-init` for more complex configurations.
- IMDSv2 (token-based) is more secure than v1; enforce v2 via instance settings.

```bash
# User data example
#!/bin/bash
yum update -y
yum install -y nginx
systemctl enable --now nginx

# IMDSv2 example
TOKEN=$(curl -X PUT "http://169.254.169.254/latest/api/token" -H "X-aws-ec2-metadata-token-ttl-seconds: 21600")
curl -H "X-aws-ec2-metadata-token: $TOKEN" http://169.254.169.254/latest/meta-data/instance-id
```

### IAM Instance Profile

- Attach IAM role to EC2; app gets temporary credentials from IMDS — no stored access keys.
- Verify: `aws sts get-caller-identity` from inside the instance.
- Always use instance profiles; never store access keys on EC2.
- Instance profiles are the standard way to give EC2 access to AWS services.

### Security Group

- Firewall rules: inbound (who can reach the instance), outbound (where it can connect).
- Common: allow SSH (22) from VPN/bastion only; allow app port from ALB SG.

### Lifecycle and Billing

| State | Compute Cost | EBS Cost | Notes |
|-------|-------------|----------|-------|
| Running | Yes | Yes | Full billing |
| Stopped | No | Yes | Private IP may change |
| Terminated | No | Root deleted | Instance gone |

- **Elastic IP**: Static public IP; free when attached to running instance.
- **Spot**: Up to 90% discount; 2-minute interruption notice; for fault-tolerant workloads.
- **Reserved/Savings Plan**: 1–3 year commitment; up to 72% discount.
- **Termination protection**: Prevent accidental termination.
- Stopped instances don't incur compute charges but EBS volumes are still billed.
- Spot instances can be reclaimed with 2-minute warning; use for batch, CI, stateless workloads.

### Placement Groups

| Type | Behavior | Use Case |
|------|----------|----------|
| Cluster | Same rack, low latency | HPC, tightly coupled |
| Spread | One instance per hardware | Critical instances |
| Partition | Groups on different hardware | HDFS, Cassandra |

# Troubleshooting Guide

### Cannot SSH into instance
1. Check Security Group: inbound TCP 22 from your IP.
2. Check key pair: correct `.pem` file, correct permissions (`chmod 400`).
3. Check instance has public IP (or connect via bastion/VPN).
4. Check OS: Amazon Linux uses `ec2-user`, Ubuntu uses `ubuntu`.

### Instance stuck in "Pending" or "Stopping"
1. Pending: capacity issue in AZ; try different AZ or instance type.
2. Stopping: EBS volume issue; wait or force-stop from CLI.
3. Check instance status checks in console.

### EBS volume full
1. Check disk: `df -h` inside instance.
2. Resize volume in AWS Console (no downtime for gp3/io2).
3. Extend filesystem: `growpart` + `resize2fs` (ext4) or `xfs_growfs` (xfs).
