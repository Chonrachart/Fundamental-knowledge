# Core Building Blocks

### Fork
- **What it is** — Your own copy of someone else's repo on github. If you don't have write permission (not a collaborator), you must fork to contribute.

### PR Anatomy
- **What it is** — A request asking repository owenr to merge your commits into their repository. Commonly used when you don't have write permission. Also used in team workflows for code review and collaboration.

#### Pull Request Workflow

1. Fork the repository (if no write access).
2. Clone your fork locally.
3. Create a new branch.
4. Make changes and commit.
5. Push the branch to your fork.
6. Open a Pull Request from your fork to the original repository.
