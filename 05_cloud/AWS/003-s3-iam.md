# S3

- Object storage; bucket (global namespace) + key (path-like); no hierarchy (prefix convention).
- **Durability**: 99.999999999%; **availability** depends on storage class and options.
- Max object 5 TB; multipart upload for large files.

# Bucket

- Container; name globally unique; choose region; block public access recommended.
- **Versioning**: Keep object versions; protect from overwrite/delete (MFA delete optional).
- **Lifecycle**: Transition to cheaper class (e.g. IA, Glacier) or expire after days.

# Storage Class

- **S3 Standard**: General purpose; high availability and throughput.
- **S3 IA**, **S3 One Zone-IA**: Infrequent access; lower cost.
- **Glacier**, **Deep Archive**: Archive; retrieval delay and cost.
- **Intelligent-Tiering**: Auto move between tiers by access pattern.

# IAM

- **User**: Identity for people or apps; long-term credentials (access key).
- **Group**: Collection of users; attach policies to group.
- **Role**: Identity with temporary credentials; assume role from user, service, or account.
- **Policy**: JSON document; allow/deny actions on resources; attach to user, group, or role.

# Policy

- **Effect**: Allow or Deny.
- **Action**: API actions (e.g. s3:GetObject, ec2:RunInstances).
- **Resource**: ARN(s) the action applies to.
- **Condition**: Optional (e.g. IP, MFA, tag).
- Prefer least privilege; use conditions to narrow.

# Role

- No long-term password; assume role to get temporary credentials.
- **EC2 instance role**: Instance assumes role; no stored key; use for app on EC2.
- **Cross-account**: Allow another account to assume role for access.
